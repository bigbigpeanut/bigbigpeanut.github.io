    一、环境变量

    环境变量是操作系统(Windows、Linux、Mac)在运行的时候,记录的一些关键性信息，用以辅助系统运行

    在Linux系统中执行:

    env命令

    即可查看当前系统中记录的环境变量

    环境变量是一种KeyValue型结构，即名称和值

    环境变量：PATH

    在前面提出的问题中，我们说无论当前工作目录是什么,都能执行/usr/bin/cd这个程序，这个就是借助环境变量中PATH这个项目的值来做到的。

    PATH环境变量十分常见

    输入

    env | grep PATH

    PATH=/usr/local/bin:/usr/bin:/usr/local/sbin:/usr/sbin:/home/itheima/.local/bin: /home/itheima/bin

    当执行任何命令，都会按照顺序，从上述路径中搜索要执行的程序的本体比如执行cd命令，就从第二个目录/usr/bin中搜索到了cd命令，并执行

    $符号（这个符号在MarkdownPad 2里面居然那么难打出来）

    在Linux系统中$符号被用于取”变量”的值

    环境变量记录的信息，除了给操作系统自己使用外，如果我们想要取用，也可以使

    用取得环境变量的值就可以通过语法:

    $环境变量名来取得比如:

    echo $PATH

    就可以取得PATH这个环境变量的值，并通过echo语句输出出来

    又或者：

    echo  ${PATH}ABC

    当和其它内容混合在一起的时候，可以通过#来标注取的变量是谁

    啊 我没怎么看自行配置环境变量的内容

    好像挺麻烦的 要输入export指令

    语法：

    export 变量名=变量值

    (这个是临时的，永久的要改文件)
