    RCE

    形成原因

    应用有时需要调用一些执行系统命令的函数，如PHP中的system、exec、shell_exec、passthru、popen、proc_popen等，当用户能控制这些函数的参数，并且开发人员对这个参数没有严格的过滤时就可以将恶意系统命令拼接到正常命令中，从而造成命令执行攻击，这就是命令执行漏洞。

    这里注意是应用需要时调用，说明应用是有权限的，没有权限就是白搭

    漏洞的分类

    1、代码层过滤不严

    商业应用的一些核心代码封装在二进制文件中，在web应用中通过system函数来调用： 

    system("/bin/program --arg $arg");

    2、系统的漏洞造成命令注入

    3、调用的第三方组件存在代码执行漏洞

    漏洞触发前提条件

    1.开发人员调用了能够执行系统命令的函数

    （就是前面说的有权限）

    2.这个函数的参数可控（即用户能够控制）

    3.开发人员没有对该函数的参数进行过滤或过滤不严

    （这里的过滤不严，其实也和个人绕过的能力相互影响的）

    eval()

    把字符串当成 PHP 代码来计算

    例子：

    <?php
$string = "beautiful";
$time = "winter";

$str = 'This is a $string $time morning!';
echo $str. PHP_EOL;

eval("\$str = \"$str\";");
echo $str;
?>

    结果：

     This is a $string $time morning!
 This is a beautiful winter morning!

    可以看看两者的区别 理解一下 把$str的内容替代输出了

    命令拼接符号

    如果说命令执行必须需要的是什么，命令拼接符号一定是第一个。

    因为恶意命令一般都是出现在正常的命令之后，需要进行符号的拼接。

    所以命令拼接符号会是你利用命令执行漏洞的基础，以下是一些一些拼接符

    window

    拼接符—-示例——-拼接符的详解

    &———-A&B——–无论A是false还是true，B都执行，即互不影响

    &&——–A&&B——-具有短路效果，A是false，B就不执行，有短路效果

    |————A|B———表示A命令语句的输出，作为B命令语句的输入执行。当A为false的时候将不会执行

    ||———–A||B———-表示A命令语句执行失败，然后才执行B命令语句

    markdown没有表格，只能用符号让他们看起来整齐咯（不对呀，我可以用图片呀，恼火）

    linux

    linux的命令有点复杂，一共有5个，下面我们一个一个介绍

    1、&

    使命令在后台运行

    只要在命令后面跟上一个空格和 ‘&’。

    你可以一口气 在后台运行多个命令。需要停止，就是kill -s 进程号，展示

    2、；

    就是分号，作用就是可以进行多条命令的无关联执行，每一条执行结果 互不影响

    3、&&

    与，左边成功才执行右边

    4、||

    或，左边失败才执行右边

    补充：单独一个管道符，就是|符号，是这样的格式：

    命令1 | 命令2

    意思是把后面命令2执行的结果，给前面命令执行1

    常见的油：

    grep | ‘cat 1.txt’ txt

    将cat到的文本内容，过滤文本txt后输出剩余文本内容

    通配符

    ？

    匹配单个字符。如果匹配多个字符，就需要多个连用

    *

    *代表任意数量的字符（这个在linux命令行中比较常用）

    [ ]

    代表一定有一个在括号内的字符(非任意字符)。例如 [abcd] 代表一定有一个字符， 可能是 a, b, c, d 这四个任何一个
    （这个经常在服务器过滤时经常用）

    一般在搜索内容和文件路径的时候用的比较多，当然当然，不仅限于同时使用单个，灵活！！

    连接符

    单双引号反斜杠 “ “ ‘ ’

    未初始变量

    未初始化的变量值都是null，例如a , a,a,b,在linux环境下输出都为null ，所以可以用未初始化变量加在一些命令的末尾，来绕过一些黑名单

    例：

    cat$a /etc$a/passwd$a
