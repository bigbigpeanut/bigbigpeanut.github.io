                                      简单介绍

    Base系列：
    Base16:就是十六进制，内容:0-9的数字和大写字母A-F，注意是F
    Base32：大写英文字母，数字234567
    Base64:大小写a-Z,数字0-9，还有+和/ 其中=是填充符
    Base85:大小写字母a-Z,数字0-9，除了+/ 还有()*,-.:;?@[]^_`{|}~
    其他Base不太常见
    特征：几乎全靠感觉，=有时是重要的判断方法

    rot系列：
    ROT编码是一种简单的码元位置顺序替换暗码。此类编码具有可逆性，可以自我解密。按照它们的ASCII值进行位置替换
    ROT5：只对数字进行编码，用当前数字往后数的第5个数字替换当前数字。例：1——>6。
    ROT13：只对字母进行编码，用当前字母往后数的第13个字母替换当前字母。例：A——>N
    ROT18：这是一个异类，本来没有，它是将ROT5和ROT13组合在一起。例：1a——>6n
    ROT47：对数字、字母、常用符号进行编码，按照它们的ASCII值进行位置替换，用当前字符ASCII值往前或后数的第47位对应字符替换当前字符。例：0——>_ ，z——K。主要特征：好像不太好看出来，排除其他的编码的特征，应该差不多是这个了，除非题目有说13之类的提示。

    栅栏密码：把文本按照一定的字数分成多个组，分多次依次取出组合成密文

    维吉利亚：

    凯撒密码：最常见的，不多说了

    摩斯密码：也很容易判断，不多说
    https://img-blog.csdnimg.cn/20190821153653265.jpg?x-oss-process=image/watermark,type_ZmFuZ3poZW5naGVpdGk,shadow_10,text_aHR0cHM6Ly9ibG9nLmNzZG4ubmV0L3dlaXhpbl80NDMwMDI4Ng==,size_16,color_FFFFFF,t_70 附个密码表吧

    MD5:这个主要是hash函数进行的，无法破译，但是可以用大数据，堆量进行解密，分为：16位和32位的，由 09，AF 组成，0x表示的十六进制数。我记得我有个网站来着：https://www.cmd5.com/

    Unicode：编码是一种力求容纳世界上所有字符的编码格式，因此也被称为万国码、统一码等等。例子：&#116;&#104;&#105;&#115;&#32;&#105;&#115;&#32;&#85;&#110;&#105;&#99;&#111;&#100;&#101;

    XXencode:感觉和unicode一样, 字符集的编码方式,特征还是比较明显的。一般是以这种形式：\xe8\x8f\xe9\xb8\x9f\xe6\x95\x99\xe7\xa8\x8b

    Quoted-printable：经常在邮件的原始信息上可以看到，是一种可打印字符编码，特征也很明显：=BC=95=E7=94=A8=E7=BC=96=E7=A0=81。给个解码网站把：http://ctf.ssleye.com/quoted.html

    培根密码（康尼密码：Bacon’s cipher）：是由法兰西斯·培根发明的一种隐写术。培根密码本质上是将二进制信息通过样式的区别，加在了正常书写之上。
    还是挺有意思的：
    https://img-blog.csdnimg.cn/20190821172631122.jpg?x-oss-process=image/watermark,type_ZmFuZ3poZW5naGVpdGk,shadow_10,text_aHR0cHM6Ly9ibG9nLmNzZG4ubmV0L3dlaXhpbl80NDMwMDI4Ng==,size_16,color_FFFFFF,t_70
    这是他的加密原理。经常以这种形式，和二进制很像，当不是二进制的时候就要考虑这个了：ABBBBAABAAABBABAABBAAABAAABBABABAAABAABAAAABAABBBAABBBAABABB

    猪圈密码：哈哈哈很有意思，直接上图
    https://img-blog.csdnimg.cn/20190821173846113.jpg?x-oss-process=image/watermark,type_ZmFuZ3poZW5naGVpdGk,shadow_10,text_aHR0cHM6Ly9ibG9nLmNzZG4ubmV0L3dlaXhpbl80NDMwMDI4Ng==,size_16,color_FFFFFF,t_70

    brainFuck编码和Ook！：比较小众，但是比较有意思
    BrainFuck 语言只有八种符号，所有的操作都由这八种符号(> < + – . , [ ])的组合来完成。
    Ook！语言是由Ook!.?组成。这两种的特征十分明显，直接上解码网站：https://www.splitbrain.org/services/ook

    波利比奥斯棋盘：棋盘密码的一种。简单的来说就是把字母排列好，用坐标的形式表现出来。字母是密文，明文便是字母的坐标。

    JSFuck编码：JSFuck是一种深奥的 JavaScript 编程风格。以这种风格写成的代码中仅使用 [、]、(、)、! 和 + 六种字符
    例：[][(![]+[])[+[]]+([![]]+[][[]])[+!+[]+[+[]]]+(![]+[])[!+[]+!+[]]+(!![]+[])[+[]]+(!![]+[])[!+[]+!+[]+!+[]]+(!![]+[])[+!+[]]][([][(![]+[])[+[]]+([![]]+[][[]])[+!+[]+[+[]]]+(![]+[])[!+[]+!+[]]+(!![]+[])[+[]]+(!![]+[])[!+[]+!+[]+!+[]]+(!![]+[])[+!+[]]]+[])[!+[]….
    有点长，给个破解网站把：http://www.jsfuck.com

    HTML实体编码：web经常看到啦，特征挺明显的，有他的预留标签”<>”，一般以&#开头

    shellcode编码:实际是一段代码（也可以是填充数据），是用来发送到服务器利用特定漏洞的代码，一般可以获取权限。shellcode编码：\x54\x68\x65，其中54、68、65为T、h、e的ASCII编码转换为二进制，再转换为16进制数。

    URL编码:也称为百分号编码（特征），URL地址（常说网址）规定了常用地数字，字母可以直接使用。
    功能强大，编码简单。
    例：%54%68%65%

    Escape/Unescape编码：%u形式， %u0054%u0068%u0065

    敲击码:基于5×5方格波利比奥斯方阵来实现.不同点是是用K字母被整合到C中。方阵：https://img-blog.csdnimg.cn/20201223174520460.png

                                     MISC图片隐写总结

    高宽的图片信息隐藏：主要是.png文件遇到的高宽修改会比较多，我用B神的工具会比较多，好用且通用，这里jpg有点特殊：https://img-blog.csdnimg.cn/ee7a29c7c87c43e78ce69e3b3a4f0470.png
    高宽的修改还有一个进阶的bmp图片格式，和深度有关，具体有些忘记了，是和图片色彩储存有关系，可以根据图片大小和深度信息来推出bmp图片是否被修改了长高。

    steghide隐写（无密码）：
    kali插件（已安装），一般用于.jpg文件。用法：steghide extract -sf 文件名

    LSB、MSB隐写（无密码）：
    我一般Stesolve使用来解LSB，最主要的特征是RGB不同通道的图片有点奇怪。
    找到相关的通道，然后保存下来。对jpg格式的图片不能使用LSB，因为jpg的储存方式不是精确储存，就不细说了。一般是png或bmp

    outguess隐写（有密码）：
    kali插件，一般用于jpg文件
    加密：
    outguess -k “密码” -d hidden.txt demo.jpg out.jpg
    加密之后，out.jpg会覆盖demo.jpg
    hidden.txt中的内容是要隐藏的东西
    解密：
    outguess -k “密码” -r out.jpg hidden.txt
    或者无密码的时候：outguess -r 文件名 hidden.txt

    exiftool图片信息查看：
    kali插件，一般用于.jpg文件，命令：exiftool cat.jpg
    https://img-blog.csdnimg.cn/59299e8cfd4c46e08acf22aa4ec8ab21.png

    zsteg隐写（无密码）：
    kali插件,一般用于bmp、png。是lsb的隐写的一键输出全部搭配

    图片容差隐写：
    一般是两张很相似的图片，但是肉眼分不出细微的差别，一般使用stegsolve
    但是也有特例，比如一下这题
    https://img-blog.csdnimg.cn/dba3cd7bb3c549d48a94b2e240f41ec4.png?x-oss-process=image/watermark,type_d3F5LXplbmhlaQ,shadow_50,text_Q1NETiBAQmV0dGVyRHJlYW0=,size_19,color_FFFFFF,t_70,g_se,x_16

    数字水印的隐写，频域盲水印隐写：这里的所有水印盲写，我都用b神的软件。主要是起到保护版权的用途：一般kali用这个： bwm decode 1.png 2.png flag.png
    https://img-blog.csdnimg.cn/3475981e4bf04528b2e2de7c71fb6175.png

    或者可以用这些：BlindWaterMark(python) 基于 python 的图像盲水印、BlindWatermark(java) 基于 java 的图像盲水印、WaterMark(隐藏水印）：图像隐写工具，在频域添加数字水印 、WaterMarkH 单图盲水印(频域隐写)工具、

    Stegdetect：很强大的隐写检测集合工具
    kali插件，一般用于检测.jpeg，搭配stegbreak食用更佳。可以检测到通过JSteg、JPHide、OutGuess、InvisibleSecrets、F5、appendX和Camouflage等。
    命令：stegdetect -s 10.0 文件名

    一些检测隐写的工具：zsteg ：PNG 和 BMP 图片隐写、StegoVeritas：隐写工具、

    关于图片的异或数据进行存写有关信息：

    stegpy：kali工具，本工具隐写后，文件会有一个下划线（简单题的时候就是一个很明显的标志），且默认生成png图片，当然可以选择-p进行密码的选择，
    命令：stegpy 隐写内容 文件名 -p

    cloacked-pixel:kali工具，一般有密码，png图片比较多见，解密的时候记得在目录里面解密，LSB类隐写
    加密：python lsb.py hide big.png 1.txt 123456
    解密：python lsb.py extract big.png-stego.png 3.txt 123456

                            MISC音频、视频隐写总结

    Steghide：没错，音频类也会遇到这个，不多介绍了

    Audacity：很强大的音频分析软件，个人感觉应该是和另一款A音频分析软件并驾齐驱的神级工具

    Mp3stego:音频隐写提取工具

    silenteye：经常称为沉默之眼，图像和音频的隐写工具

    DeepSound：有密码，可以将文件加密保存到一段声音文件中

    Mp3tag：音频文件元资料编辑器
